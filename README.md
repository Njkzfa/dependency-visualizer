Что делает инструмент  
CLI‑утилита строит и анализирует граф зависимостей пакетов python(pip) без использования готовых средств менеджера пакетов и сторонних библиотек получения зависимостей.  

Этап 1. Параметры из опций командной строки; печать всех параметров (ключ‑значение); версия пакета; имя сгенерированого файла с изображением.  
Этап 2. Получение прямых зависимостей пакета из Python (pip) выбранного репозитория (URL/путь) и вывод на экран.  
Этап 3. Построение графа DFS c рекурсией, корректная обработка циклов; поддержан тестовый режим (файл с графом, имена пакетов — большие латинские буквы).  
Этап 4. Вывод порядка загрузки (PySocks; certifi, chardet и др.).  
Этап 5. Генерация текста диаграммы на языке Mermaid; сохранить файл в формате SVG.

Установка и запуск  
Зависящих от менеджера пакетов библиотек нет. Для реальных репозиториев python должен быть доступен по URL/пути.  
  
# Запуск из исходников  
python main.py --help  
Основные действия:  

# Этап 1: печать конфигурации  
python main.py configs/config.xml  
python main.py configs/config.json  
  
# Этап 2: прямые зависимости  
python main.py direct -p <PKG> -r <REPO_OR_FILE> [-t]  
  
# Этап 3: подграф достижимых узлов (DFS, с рекурсией)  
python main.py graph  -p <PKG> -r <REPO_OR_FILE> [-t]
  
# Этап 4: порядок загрузки (топосорт по подграфу)  
python main.py order  -p <PKG> -r <REPO_OR_FILE> [-t]  
  
# Этап 5: Mermaid (печать и/или запись в файл)  
python main.py mermaid -p <PKG> -r <REPO_OR_FILE> [-t] -o graph.mmd
Параметры  
-p/--package — имя анализируемого пакета.  
-r/--repo — путь/URL к python(pip). В тестовом режиме — путь к текстовому файлу с графом.
-t/--test — включить тестовый режим (файл‑граф) 
-o/--out — путь для сохранения Mermaid‑текста при action=mermaid.  
Формат тестового репозитория  
Файл содержит строки вида:    
  
A: B C  
B: C  
C:  
Поддерживается также запись A -> B, C. Имена пакетов — большие латинские буквы, как в задании.  
  
Как работает парсер Python пакетов (без «готовых средств»)
1. Вызов системной команды
pip show <package_name>
2. Парсинг вывода:
Построчный анализ вывода команды
3. Извлечение данных:
Поиск строки Requires и разбор зависимостей
Обработка ошибок:
Коректная обработка отсутствующих пакетов
